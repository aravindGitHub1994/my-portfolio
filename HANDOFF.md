## Current Status
- **Slices 1–6 are implemented and passing `npm run lint` + `npm run build`**
  (zero errors/warnings). Two items remain, both below in Unresolved Threads:
  generating the four `*.light.svg` diagram assets, and the manual browser visual
  pass. Nothing is committed yet.
- **Slice 1 — theme pipeline.** Tailwind v4 emits `var(--color-*)` references
  (confirmed in built CSS), so the `html[data-theme="day"]` token block flips the
  whole site with zero component edits. Day "Warm sunlit" palette is AA-verified;
  amber accent darkened to `#82661b` (4.61:1 on cream).
  - `src/lib/theme.ts` — single source of truth: `ThemeMode`/`ResolvedTheme`,
    `DAY_START_HOUR=6`/`DAY_END_HOUR=18`, `resolveTheme()`, `themeInitScript`.
  - `src/components/ThemeProvider.tsx` — client provider `{ mode, resolved, setMode }`;
    re-resolves auto on `focus`/`visibilitychange` only (no live mid-session flip).
  - `src/app/globals.css` — `html[data-theme="day"]` token block + day wash.
  - `src/app/layout.tsx` — pre-paint `<script>` in `<head>`, `suppressHydrationWarning`.
- **Slice 2 — "alive" switcher.** `src/components/ThemeToggle.tsx` (NEW) replaces the
  deleted `ThemeToggleTemp.tsx`. Auto "A" badge button (`aria-pressed`) + scene
  button (moon at night / sun at day). `useSyncExternalStore` mount gate → neutral
  ring icon on SSR (no hydration mismatch). Clicking scene sets explicit mode +
  clears auto; "A" enables auto (scene then tracks clock, shows `a` overlay).
  Reduced-motion safe. Lives in `Header.tsx` right cluster, desktop + mobile.
- **Slice 3 — day living background.** `src/components/Cloudfield.tsx` (NEW, sun +
  drifting clouds, sky→cream gradient, reduced-motion static) and
  `src/components/BackgroundScene.tsx` (NEW client wrapper) gate exactly one canvas
  per resolved theme (night→`Starfield`, day→`Cloudfield`); `null` on SSR.
  `layout.tsx` now renders `<BackgroundScene />` instead of `<Starfield />`.
- **Slice 4 — glow token.** `--color-glow` added to `@theme` (gold) with a day
  override (soft amber). The 5 hardcoded `rgba(212,175,55,…)` glows
  (`Button.tsx`, `Hero.tsx`, `ProjectCard.tsx`, `ProjectModal.tsx`, `OrbitRing.tsx`)
  now use `var(--color-glow)`. `CursorSpotlight.tsx` reads `useTheme()` and is
  disabled in day mode. `.text-gilt` day shimmer comes free via the token flip.
- **Slice 5 — light diagrams DONE.** `ProjectModal.tsx` computes `diagSrc =
  resolved === "day" ? project.diagram.replace(".svg", ".light.svg") :
  project.diagram`. The four `public/diagrams/*.light.svg` assets are generated by
  `scripts/diagrams-light.js`, which remaps the committed dark SVGs to the day
  tokens (same geometry, no layout drift) — re-run `node scripts/diagrams-light.js`
  after editing any source diagram. No `mmdc`/Chromium dependency needed.
- **Slice 6 — a11y + docs.** `src/components/ThemeMetaColor.tsx` (NEW) updates the
  `theme-color` meta on theme change; `layout.tsx` exports
  `viewport: Viewport = { themeColor: "#07071a" }` (Next 16 API, not the deprecated
  `metadata.themeColor`). `docs/design-system.md` rewritten for dual-theme, links ADR-003.
